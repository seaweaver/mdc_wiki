# 蚂蚁渠道基金产品提报监控｜数据加工层

## 概览

本页是"蚂蚁渠道基金产品提报监控"项目**数据加工层**的入口页，与业务规则层 `[[rules/蚂蚁选品指标框架]]` 形成上下两层对应：

- **上层（业务规则层）**：定义"要判断什么"——选品指标、赛道阈值、预警分级
- **下层（数据加工层）**：定义"指标怎么算出来"——物理表结构、计算规则、加工流程

## 数据加工总览

整个加工链路的核心逻辑：

```
基金池入口表 (TMP_JKQD_FUND_METRICS)
  → 13个 INSERT 步骤写入窄表 (T_DM_JKQD_FUND_METRICS)
    → 6个 MERGE 批次聚合到宽表 (DM_JKQD_FUND_METRICS)
```

- **宽表** `DM_JKQD_FUND_METRICS`：蚂蚁渠道监控指标结果表，89个字段，最终对外输出
- **窄表** `T_DM_JKQD_FUND_METRICS`：中间工作表，13个步骤分批写入，保留最近30天
- **基金池入口表** `TMP_JKQD_FUND_METRICS`：控制哪些基金需要计算指标（S_FROM_PLACE='1'）
- **债券久期中间表** `TMP_BOND_LAST_DURATION`：组合久期计算的关键中间表
