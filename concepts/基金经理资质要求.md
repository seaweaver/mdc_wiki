# Manager Qualification Requirements｜基金经理资质要求

## 定义｜Definition

基金经理资质要求，是蚂蚁选品中用于判断“当前基金经理是否足以支撑产品进入目标赛道”的一组经验门槛集合。

它不是单一字段，而是由多种经验维度共同构成。

## 包含的核心维度｜Core Dimensions

### 1. 投资年限

- 关注基金经理累计从事投资管理相关工作的年限
- 常见门槛：
  - 求赚纯债短债 / 中短债：`>2年`
  - 中长债、求赚固收、FOF：`>3年`

### 2. 公募经验

- 更强调基金经理在公募体系内的实际管理经验
- 常见门槛：
  - 超额宝月月：`>2年`
  - 超额宝季季 / 年年、增益宝：`>3年`

### 3. 管理当前产品时长

- 关注当前基金经理在被提报产品上的任职稳定性
- 常见门槛：
  - 求赚纯债 / 求赚固收 / 定开：`>5个月`
  - 超额宝：`>1年` 或 `>2年`
  - 增益宝：`>2年`

### 4. 多资产类型产品经验

- 主要出现在 FOF 场景
- 当前已知门槛：
  - FOF：管理多资产类型产品经验 `>2年`
  - 经验范围可含 `公募 / 专户 / 投顾`

## 为什么重要｜Why It Matters

蚂蚁选品并不只看产品历史表现，也关心“当前负责这只产品的人是否足够成熟”。

因此，这组门槛实际上是在控制两类风险：

- 产品历史是否足够可归因于当前经理
- 基金经理是否具备与该赛道相匹配的投资管理经验

## 当前团队理解｜Current Team Understanding

这几个维度彼此相关，但不能互相替代：

- 投资年限：更偏总经验
- 公募经验：更偏公募产品管理经验
- 管理当前产品时长：更偏当前产品维度的稳定记录
- 多资产类型产品经验：更偏 FOF 场景的资产配置复杂度

## 易错点｜Gotchas

- 不能把“投资年限”和“公募经验”混为一谈
- 不能把“管理当前产品时长”误当成产品本身的成立时长
- FOF 的多资产经验要求比普通赛道更宽，不能直接套用其他赛道口径
- 第二层实现前，人员履历、任职起始日、产品映射关系都还需要单独补齐
