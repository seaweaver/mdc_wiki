## R17 含权仓位计算

- rule_id: R17
- local_id: qudaojiankong:R17
- content_hash: `84cc4a3d1750167f`
- use_cases: 步骤11

### 业务定义

股票仓位 + 可转债仓位×0.5，除以总资产。

### SQL 逻辑模板

```sql
(STOCK_VALUE + CONVERT_BOND_VALUE * 0.5) / NULLIF(TOTAL_ASSET, 0) * 100 AS EQUITY_POS_PCT
```

## R18 过去4个季报平均含权仓位

- rule_id: R18
- local_id: qudaojiankong:R18
- content_hash: `7cdb37e5a7e48518`
- use_cases: 步骤11

### 业务定义

取最近4个季报（0331/0630/0930/1231）的含权仓位，求平均。

### SQL 逻辑模板

```sql
SELECT FUND_CODE, AVG(EQUITY_POS_PCT) AS AVG_EQUITY_POS_4Q_PCT
FROM (SELECT ...,
      ROW_NUMBER() OVER (PARTITION BY FUND_CODE ORDER BY RPT_ENDDATE DESC) AS RN_LAST
      WHERE SUBSTR(RPT_ENDDATE,5,4) IN ('0331','0630','0930','1231'))
WHERE RN_LAST <= 4
GROUP BY FUND_CODE
```

## R19 2025年以来含权仓位字符串

- rule_id: R19
- local_id: qudaojiankong:R19
- content_hash: `0758129d94bc5b95`
- use_cases: 步骤11

### 业务定义

2025年以来各季度的含权仓位，格式: YYYYQn:xx.xx%,...

### SQL 逻辑模板

```sql
LISTAGG(SUBSTR(RPT_ENDDATE,1,4) || 'Q' ||
  CASE SUBSTR(RPT_ENDDATE,5,2) WHEN '03' THEN '1' WHEN '06' THEN '2'
       WHEN '09' THEN '3' WHEN '12' THEN '4' END
  || ':' || TO_CHAR(ROUND(EQUITY_POS_PCT,2), 'FM9990D00') || '%', ',')
  WITHIN GROUP (ORDER BY RPT_ENDDATE)
```

## R20 组合久期计算

- rule_id: R20
- local_id: qudaojiankong:R20
- content_hash: `ade1a6491c7d9b1f`
- related_tables: `[[tables/中国债券衍生指标-CBondValuation]]`, `[[tables/债券久期中间表-TMP_BOND_LAST_DURATION]]`
- use_cases: 步骤11（依赖 TMP_BOND_LAST_DURATION 中间表）

### 业务定义

以最新季报中债券市值为权重，对单券久期加权平均。

### 口径要点

- 单券久期来源: CBondValuation.B_ANAL_DURATION（取季报末前15天窗口内最新估值）
- 权重: BondPortfolio.F_PRT_BDVALUE（债券市值）
- 只计算久期非空且市值>0的债券

### SQL 逻辑模板

```sql
SUM(DURATION * F_PRT_BDVALUE) / NULLIF(SUM(F_PRT_BDVALUE), 0) AS PORT_DURATION
```

## R21 持有期类型

- rule_id: R21
- local_id: qudaojiankong:R21
- content_hash: `7dbe257fa59236b0`
- related_tables: `[[tables/持有期封闭期配置-MD_PROD_CXQ_CFG]]`, `[[tables/PLMS-基金竖表字段-PLMS2_TFUNDINFOVERTICAL]]`, `[[tables/PLMS-公募基金信息-PLMS2_TPDTFUNDINFO]]`
- use_cases: 步骤12

### 业务定义

- **HOLD_PERIOD_TYPE**: 持有期类型，来源于 CJHX_SMI.MD_PROD_CXQ_CFG
- **CLOSED_PERIOD_TYPE**: 封闭期类型，来源于 PLMS2 系统 C_OPERATIONTIME 字段（值+'M'）
- 两者同时为空时代表全开放式

### SQL 逻辑模板

```sql
-- 持有期: 从配置表取
SELECT UPPER_BK_PRODUCT, FUND_HOLD_PERIOD AS HOLD_PERIOD
FROM CJHX_SMI.MD_PROD_CXQ_CFG
-- 封闭期: 从PLMS2竖表字段取
SELECT A.C_TAFUNDCODE, C.C_CONTENT || 'M' AS CLOSED_PERIOD
FROM STAGE.PLMS2_TPDTFUNDINFO A
INNER JOIN STAGE.PLMS2_TFUNDINFOVERTICAL C
  ON A.C_FUNDCODE = C.C_FUNDCODE AND C.C_PROCODE = 'C_OPERATIONTIME'
WHERE C.C_CONTENT <> '0'
```
