## R01 业务日期与数据窗口

- rule_id: R01
- local_id: qudaojiankong:R01
- content_hash: `ac5bed826a74c594`
- related_tables: `[[tables/交易日历-MD_PUB_DATE_INFO]]`

### 业务定义

跑批日期默认为 T-1，通过 `v_buss_date_8` (YYYYMMDD) 和 `v_buss_date` (YYYY-MM-DD) 控制。

### 适用场景

所有步骤的数据过滤基准日。

### SQL 逻辑模板

```sql
-- v_today = 'auto' 或 '99991231' 时取 sysdate-1
-- 同时从 STAGE.MD_PUB_DATE_INFO 取最近交易日
WHERE data_date <= v_buss_date_8
```

## R02 基金池过滤

- rule_id: R02
- local_id: qudaojiankong:R02
- content_hash: `34b6580f6093a266`
- related_tables: `[[tables/基金池入口表-TMP_JKQD_FUND_METRICS]]`
- related_dictionary: `基金池标记 (S_FROM_PLACE)`

### 业务定义

多数计算只针对 `S_FROM_PLACE = '1'` 的基金。

### 适用场景

涉及 Wind 净值/业绩/持仓等计算的步骤（步骤2~11）。

### SQL 逻辑模板

```sql
INNER JOIN TMP_JKQD_FUND_METRICS E
  ON source.F_INFO_WINDCODE = E.PRODUCT_CODE
  AND E.S_FROM_PLACE = '1'
```

## R24 产品代码格式

- rule_id: R24
- local_id: qudaojiankong:R24
- content_hash: `d723aea142b9e52c`

### 业务定义

产品代码统一为 Wind 格式，6位基金代码 + '.OF' 后缀。

### 适用场景

所有步骤。

### SQL 逻辑模板

```sql
BK_PRODUCT || '.OF' AS PRODUCT_CODE
```
