## R22 封闭期正收益概率

- rule_id: R22
- local_id: qudaojiankong:R22
- content_hash: `64c4dbdd686005c3`
- use_cases: 步骤13

### 业务定义

每个封闭期内，期初净值到期末净值的收益率>0的概率。

### 口径要点

- 期初净值: <= OPR_STARTDT 的最后一期净值（若缺失则取 >= OPR_STARTDT 的第一期）
- 期末净值: <= OPR_ENDDT 的最后一期净值
- 净值: NVL(F_NAV_ADJUSTED, F_NAV_UNIT)
- 统计范围: 近2年（24个月）/近3年（36个月）内结束的封闭期

### SQL 逻辑模板

```sql
-- 期初净值取法 (优先取≤起始日的最后一期，缺失时回退≥起始日的第一期)
NVL(MAX(CASE WHEN PRICE_DATE <= OPR_STARTDT THEN NAV END)
      KEEP (DENSE_RANK LAST ORDER BY CASE WHEN PRICE_DATE <= OPR_STARTDT
        THEN PRICE_DATE ELSE '00000000' END),
    MIN(CASE WHEN PRICE_DATE >= OPR_STARTDT THEN NAV END)
      KEEP (DENSE_RANK FIRST ORDER BY CASE WHEN PRICE_DATE >= OPR_STARTDT
        THEN PRICE_DATE ELSE '99999999' END)) AS NAV_START
-- 正收益概率
ROUND(100 * SUM(CASE WHEN PERIOD_RETURN > 0 THEN 1 ELSE 0 END)
            / NULLIF(SUM(CASE WHEN PERIOD_RETURN IS NOT NULL THEN 1 ELSE 0 END), 0), 4)
```

## R23 窄表→宽表 MERGE 聚合规则

- rule_id: R23
- local_id: qudaojiankong:R23
- content_hash: `217a1af76a0d7bd0`
- use_cases: 6 个 MERGE 批次

### 业务定义

窄表按 PRODUCT_CODE + DATA_DATE 聚合到宽表。

### 聚合规则

- 数值型字段 (收益率、概率、规模等): `SUM`（窄表中每产品每批只有一行有效值）
- 字符串型字段 (排名串、产品列表等): `MAX`（同理只有一行有效值）

### MERGE 流程

1. 先 UPDATE 宽表对应字段为默认值 (0 / NULL / SYSDATE)
2. 再 MERGE INTO (MATCHED → UPDATE, NOT MATCHED → INSERT)
3. 关联条件: `NVL(A.PRODUCT_CODE,'-') = NVL(B.PRODUCT_CODE,'-') AND A.DATA_DATE = v_buss_date_8`

具体聚合字段映射见 `tables.md` 第8节。
